协作者管理 Tab 怎么设计？

添加协作者、修改角色、转让所有权的交互方式

A

内联列表 + 行内操作

协作者 Tab 直接展示用户列表，每行显示头像/名称/角色，角色通过下拉框直接修改，删除按钮在行末

顶部 "添加协作者" 展开内联表单（输入用户 ID + 选择角色），底部 "转让所有权" 按钮

│ [编辑器] [文件] [版本历史] [协作者]    │
│ ────────────────────────────────────    │
│ [+ 添加协作者]                       │
│ ────────────────────────────────────    │
│ 👤 Alice (owner)                   │
│ 👤 Bob    [editor ▾]          [✕]  │
│ 👤 Carol  [viewer ▾]          [✕]  │
│ ────────────────────────────────────    │
│                [🔄 转让所有权]     │

✓ 操作直观，无弹窗  ✓ 角色修改即时生效  ✓ 与版本历史 Tab 风格一致

B

列表展示 + Dialog 操作

协作者 Tab 展示只读列表，所有操作（添加、改角色、删除、转让）通过点击按钮弹出 Dialog 完成

更适合需要确认的破坏性操作（删除、转让），但操作步骤多一层

│ [编辑器] [文件] [版本历史] [协作者]    │
│ ────────────────────────────────────    │
│ [+ 添加]                            │
│ ────────────────────────────────────    │
│ 👤 Alice   owner                   │
│ 👤 Bob     editor         [⋯ 菜单]  │
│ 👤 Carol   viewer         [⋯ 菜单]  │
│                                    │
│                [🔄 转让所有权]     │

✓ 破坏性操作有确认步骤  △ 改角色需两步  △ 弹窗较多
