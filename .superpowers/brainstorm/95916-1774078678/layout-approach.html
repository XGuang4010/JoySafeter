三大功能的 UI 入口放在哪里？

版本管理 & 协作者管理属于 Skill 级别，Token 管理属于用户级别。如何组织？

A

Skill 详情内嵌 Tab + Settings 页面

版本 & 协作者：在 SkillsManager 右侧编辑区域新增 Tab（如 "版本历史"、"协作者"），与现有文件编辑器平级切换

Token：放到全局 Settings 对话框中新增一个 "API Tokens" 页签

✓ 版本/协作者紧贴 Skill 上下文  ✓ Token 在全局设置中统一管理

B

Skill 右键/菜单弹窗 + Settings 页面

版本 & 协作者：在 Skill 卡片或标题栏添加操作菜单（⋯），点击弹出 Dialog/抽屉来管理版本和协作者

Token：同方案 A，放到全局 Settings

✓ 不改变现有编辑区布局  △ 弹窗层级较深

C

Skill 详情侧边面板 + Settings 页面

版本 & 协作者：在编辑器右侧增加可折叠的侧边面板（类似 IDE 的属性面板），展示版本历史和协作者列表

Token：同方案 A，放到全局 Settings

✓ 可同时查看代码和版本  △ 横向空间压力大（已有三栏）
