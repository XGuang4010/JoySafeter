版本发布的入口和流程怎么设计？

用户从哪里触发"发布版本"？版本历史怎么展示？

A

顶部工具栏 "发布" 按钮 + 版本历史 Tab

在 Skill 编辑器顶部（Save 按钮旁边）添加 "Publish" 按钮，点击弹出 Dialog 填写版本号和 release notes

"版本历史" Tab 内展示时间线列表，每条可展开查看详情、恢复 draft 或删除

┌─────────────────────────────────────────┐
│ Skill Name     [Save] [Publish ▾]      │
├─────────────────────────────────────────┤
│ [编辑器] [文件] [版本历史] [协作者]   │
│ ───────────────────────────────────     │
│ v1.2.0  2026-03-20  by Alice         │
│   新增 X 功能     [恢复] [删除]      │
│ v1.1.0  2026-03-15  by Bob          │
│ v1.0.0  2026-03-10  by Alice         │
└─────────────────────────────────────────┘

✓ 发布入口醒目  ✓ 版本历史清晰  ✓ 操作路径短

B

版本历史 Tab 内置发布入口

不在顶部加按钮，发布入口放在 "版本历史" Tab 顶部，点击 "发布新版本" 展开内联表单（版本号 + notes）

版本列表在下方，同样支持恢复和删除

┌─────────────────────────────────────────┐
│ Skill Name     [Save]                 │
├─────────────────────────────────────────┤
│ [编辑器] [文件] [版本历史] [协作者]   │
│ ───────────────────────────────────     │
│ [+ 发布新版本]                      │
│ ┌ 版本号: [1.3.0]                  │
│ │ Notes: [___________]              │
│ └                [发布]            │
│ ───────────────────────────────────     │
│ v1.2.0  2026-03-20                 │
│ v1.1.0  2026-03-15                 │
└─────────────────────────────────────────┘

✓ 顶部栏不拥挤  ✓ 发布和历史同一视图  △ 需先切到 Tab 才能发布
