Token 管理页面怎么设计？

放在全局 Settings 对话框中的 "API Tokens" 页签

A

卡片列表式

每个 Token 一张卡片，显示名称、prefix、scopes 标签、过期时间、最后使用时间。右上角 "Revoke" 按钮

顶部 "Create Token" 按钮弹出 Dialog（填名称、选 scopes、设过期时间），创建成功后弹出明文 token 供复制

│ Settings > API Tokens                 │
│ ────────────────────────────────────    │
│ [+ Create Token]                     │
│                                    │
│ ┌───────────────────────────────┐     │
│ │ CI deploy token      [Revoke] │     │
│ │ sk_a1b2c3d4                 │     │
│ │ [read] [write] [publish]     │     │
│ │ Expires: 2026-12-31          │     │
│ │ Last used: 2h ago            │     │
│ └───────────────────────────────┘     │
│ ┌───────────────────────────────┐     │
│ │ Local dev token      [Revoke] │     │
│ │ sk_x9y8z7w6                 │     │
│ │ [read]                      │     │
│ │ No expiry                   │     │
│ │ Last used: never            │     │
│ └───────────────────────────────┘     │

✓ 信息丰富  ✓ 与 GitHub Personal Access Token 页面风格类似

B

紧凑表格式

Token 以表格展示（名称、prefix、scopes、过期、最后使用、操作列），更紧凑，适合 token 数量多的场景

│ Settings > API Tokens                 │
│ ────────────────────────────────────    │
│ [+ Create Token]                     │
│                                    │
│ Name        Prefix   Scopes  Exp  Act │
│ ───────────────────────────────────    │
│ CI deploy   sk_a1..  r/w/p  12/31 [⌫] │
│ Local dev   sk_x9..  r     —    [⌫] │
│ Staging     sk_m3..  r/w   06/30 [⌫] │

✓ 紧凑  ✓ 一屏看更多  △ 信息密度高，scopes 需缩写
